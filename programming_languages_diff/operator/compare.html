オブジェクトの比較演算子 2013/09/17

-> 文字列の比較
-> 数値の比較
-> 配列の比較

Java 2013/10/15

==, != はプリミティブ型の場合は値を比較し、参照型の場合は同じインスタンスかどうかを判別する。

オブジェクトに対して obj1.equals(obj2) を使うと、値を比較するが、各クラスの equals の実装に依存する。

Javaにおける == はRubyにおける equal? に似ていて、 Javaにおける equals はRubyにおける == に似ている。

Scala 2013/07/24

== は等値の場合に true を返す。 != はその反対。プリミティブ型に対してはJavaと同じで、参照型に対してはJavaの equals を呼び出している。

Ruby 2013/08/19

== は等値の場合に true を返す。 != はその反対。

equal?メソッドは同じインスタンスかどうかを判別する。

Rubyにおける == はJavaにおける equals に似ていて、 Rubyにおける equal? はJavaにおける == に似ている。

Perl 2013/07/31

eq は文字列として等値の場合に '1' を返し、そうでない場合は、 '' を返す。 ne はその反対。

== は数値として等値の場合に '1' を返し、そうでない場合は、 '' を返す。 != はその反対。

'10a' == '10b' は無理やり数値にして、同じ 10 になるため、1 を返す。ただ、 use warnings; を使っていると、警告を表示してくれる。
