キーで値を参照するには (get)

Java 2013/09/05

map.get("a");

要素が存在しない場合は null が返される。しかし、null が返されることは必ずしも要素が存在しない場合とは限らず、値として null が入っている場合もあるので注意。

Scala

要素が存在しない場合に参照時に例外をスローさせるには 2013/09/23

// Map[A, B]
def apply(key: A): B

例

map("a")

この方法だと要素が存在しない場合は NoSuchElementException がスローされる。

Option で取得するには 2013/09/23

Option で取得するには

map.get("a")

getは、要素が存在しない場合はNoneが返され、存在する場合は値をSomeで包んだ値が返される。

要素が存在しない場合にデフォルト値を返させるには 2013/09/23

要素が存在しない場合のデフォルト値をその場で設定して、要素を取得するには

// Map[A, B]
def getOrElse(key: A, default: => B): B

例

map.getOrElse("a", 0);

2つ目の引数がデフォルト値となる。要素が存在する場合には、2つ目の引数が評価されないので、計算に時間のかかるデフォルト値を設定してもよい。

PHP 2013/09/03

arr['a']

Python 2013/10/04

dic['a']

Ruby 2013/09/04

hash['a']

または

hash.fetch('a')

存在しない要素を取得しようとすると [] は nil が返ってくるが、 fetch は KeyError の例外が発生する。

fetch は第2引数かブロックを与えることができ、その場合は、要素が存在しない場合に、第2引数を返すか、ブロックを評価した結果を返す。

例

m = {'a' => 1}

p m.fetch('a') # => 1
p m.fetch('a', 2) # => 1
p m.fetch('b', 2) # => 2
p m.fetch('b') { 2 } # => 2

Perl 2013/09/10

以下のように {} でキーを指定し、先頭は $ を使う。

my %hash = (a => 1, b => 2);

$hash{a} # => 1
$hash{'a'} # => 1

存在しない要素を取得しようとすると undef になり、 use warnings; が宣言されていれば、警告が表示される。

ハッシュへのレファレンスから要素を取り出すには

my $hash = {a => 1, b => 2};

$hash->{a} # => 1
