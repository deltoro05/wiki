マップ(連想配列)書き方

Java 2013/08/01

マップは一発では作れない。

import java.util.HashMap;
HashMap<String, Integer> map = new HashMap<String, Integer>();
map.put("a", 1);
map.put("b", 2);

キーで参照するには

map.get("a");

空のマップを作るには

Collections.<String, Integer>emptyMap()

要素が1つだけのマップを作るには

Collections.<String, Integer>singletonMap(key, value)

Scala 2013/05/06

map = Map("a" -> 1, "b" -> 2);

これは以下のシンタックスシュガー。

map = scala.collection.immutable.Map.apply(scala.Predef.any2ArrowAssoc("a").->(1),
                                           scala.Predef.any2ArrowAssoc("a").->(2));

キーで参照するには

map("a")

Option で取得するには

map.get("a")

PHP 2013/05/08

arr = array('a' => 1, 'b' => 2);

PHP 5.4からは以下のようにも書ける。

arr = ['a' => 1, 'b' => 2];

キーで参照するには

arr['a']

Perl 2013/07/31

ハッシュそのものは変数名が % で始まり、要素を参照するには $ で始める。

以下の2つはまったく同じ意味で、どちらもハッシュを %hash に代入している。

my %hash = (a => 1, b => 2);
my %hash = ('a', 1, 'b', 2);

上記の (...) の表記をリストという。偶数個の要素のリストを % で始まる変数に代入するとハッシュになる。

リストの中の => は , とまったく同じ意味で、以下のように書いても差支えない。こんな書き方は普通はしないが。

my %hash = ('a', 1 => 'b', 2);

最後に余計なカンマがあっても問題ない。

my %hash = (
           a => 1,
           b => 2,
          );

キーで要素を参照するには、以下のように {} でキーを指定し、先頭は $ を使う。

$hash{a} # => 1
$hash{'a'} # => 1

ハッシュへのレファレンス 2013/07/31

my $hash = {a => 1, b => 2};

で、これは以下と同じ。

my %h = (a => 1, b => 2);
my $hash = \%m;

レファレンスの実体を取り出すには、

%$hash;

map = {'a': 1, 'b': 2}
