例外処理

書き方

Java 2013/06/25

try {
  ...
  throw ...
  ...
} catch (FileNotFoundException e) {
  ...
} catch (IOException e) {
  ...
} finally {
  ...
}

Java7から使える try-with-resources 構文では以下のように書ける。finally節を書かなくても try のカッコの中で生成したオブジェクトを確実にcloseしてくれる。

try (FileInputStream in = new FileInputStream(...)) {
  ...
} catch (FileNotFoundException e) {
  ...
} catch (IOException e) {
  ...
}

Scala 2013/07/05

try {
  ...
  throw ...
  ...
} catch {
  case e: IOException => ...
  case e: FileNotFoundException => ...
  case e => ...
} finally {
  ...
}

Ruby 2013/08/04

begin
  ...
  raise ...
  ...
rescue
  ...
else
  ...
ensure
  ...
end

rescue で例外オブジェクトを利用したい場合は、以下のように書く。

rescue => e

エラーメッセージは、 e.message で取得できる。

rescue で特定のクラスの例外を補足したい場合は、以下のように書く。

rescue ZeroDivisionError => e

rescue 節は複数書くことができる。

例外をスローするには、以下のような書き方がある。

raise "exception message ..."
raise ZeroDivisionError, "message ..."

Python 2013/04/17

try:
  ...
  raise "message ..."
  raise HogeException()
  ...
except:
  ...
finally:
  ...

except で例外オブジェクトを扱う場合は、 except ex: のように書く。

特定の例外クラスだけ except で扱う場合は、 except HogeException: または except HogeException, ex: のように書く。

except の中で同じ例外を再スローするには raise とだけ書く。

JavaScript 2013/04/16

try {
  ...
  throw "message ...";
  ...
} catch(ex){
  ...
} finally {
  ...
}

CoffeeScript 2013/04/16

try
  ...
  throw("message")
  ...
catch ex
  ...
finally
  ...

チェック例外と非チェック例外

Java 2013/07/18

チェック例外をスローする可能性のあるメソッドには、メソッドシグニチャにthrowsが必要。チェック例外は Exception でかつ RuntimeException でないもの。非チェック例外は Error または RuntimeException。

Scala 2013/07/18

Javaのチェック例外も含めすべての例外が非チェック例外の扱いになる。
