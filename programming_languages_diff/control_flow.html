if文

C言語, C++ 2013/06/19

if(...){
  ...
} else if(...){
  ...
} else {
  ...
}

本体の文が1つだけの場合は {} を省略可。

if(...)
  ...
else
  ...

いずれの構文でも else で実行する文がなければ、else を省略可。

Go言語 2013/06/19

if ... {
  ...
} else if ... {
  ...
} else {
  ...
}

本体の文が1つだけの場合でも {} を省略不可。

else で実行する文がなければ、else を省略可。

条件を書くの部分にセミコロンで区切って変数初期化の文を書くことができる。

if a := f(); a < 0 {
  return 0
} else {
  return a
}

PHP 2013/06/19

if(...){
  ...
} else if(...){
  ...
} else {
  ...
}

else if はつなげて elseif でも可。

本体の文が1つだけの場合はC言語と同様に {} を省略可。

if(...)
  ...
else
  ...

コロンを使った以下の記法も可能。

if(...):
  ...
elseif(...):
  ...
else:
  ...
endif;

いずれの構文でも else で実行する文がなければ、else を省略可。

Ruby 2013/06/01

if ... then
  ...
elsif ... then
  ...
else
  ...
end

then は省略可能。

sh 2013/04/10

if [ ... ]; then
    ...
elif [ ... ]; then
    ...
else
    ...
fi

CoffeeScript 2013/04/10

if ... then ... else ...

if ...
    ...
else if ...
    ...
else
    ...

elseだけを1行にまとめることができる。

if ...
    ...
else if ...
    ...
else ...

3項演算子としても使える。

a = if ... then ... else ...

Rubyの用に修飾子としても使える。

alert "hoge" if condition

while文

while(...){
  ...
}

本体の文が1つだけの場合は {} を省略可。

while(...)
  ...

永久ループは

for(;;){
  ...
}

Go言語 2013/06/21

for ... {
  ...
}

本体の文が1つだけの場合でも {} を省略不可。

永久ループは

for {
  ...
}

Ruby 2013/06/21

while ... do
  ...
end

do は省略可能。

永久ループは

while true do
  ...
end

C言語, C++ 2013/06/21

for(int i = 0; i < 10; i++){
    ...
}

以下の2つは同じ意味になる。

for(式1; 式2; 式3){
    ...
}

または

式1;
while(式2){
    ...
    式3;
}

Ruby 2013/06/21

0 から n-1 までを繰り返すには

n.times do |i|
  ...
end

または

for i in 0..(n-1) do
  ...
end

switch文

C言語, C++ 2013/05/15

switch(x){
case 1:
  ...
  break;
case 2:
  ...
  break;
default:
  ...
  break;
}

break を書かないとswitch文から脱出せずに次のラベルを引き続き実行してしまう。(fall through)

Java 2013/05/15

switch(x){
case 1:
  ...
  break;
case 2:
  ...
  break;
default:
  ...
  break;
}

break を書かないとswitch文から脱出せずに次のラベルを引き続き実行してしまう。(fall through)

x には、byte, short, char, int, Byte, Short, Character, Integer, enum, String が使える。

String が使えるのは、Java SE 7 から。ただ、String の場合、 null だと NullPointerException が発生してしまい、caseで null を指定したり、 defaultに飛んでくれたりはしない。 Integer の場合も、NullPointerException が発生しうるのは同様。

Byte, Short, Character, Integer が使えるのは、たぶん autoboxing が導入された Java SE 5 から。

Scala 2013/05/15

x match {
  case 1 =>
    ...
  case 2 =>
    ...
  case _ =>
    ...
}

C言語やJavaのように　fall through することはない。

Ruby 2013/06/05

case x
when 1
  ...
when 2
  ...
else
  ...
end
