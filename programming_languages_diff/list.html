リスト(配列)の書き方

Java

配列

arr = {1, 2, 3};
arr = new int[]{1, 2, 3};

インデックスで参照するには

arr[index];

Scala

arr = Array(1, 2, 3);
list = List(1, 2, 3);

これは以下と同じ意味。

arr = scala.Array.apply(1, 2, 3);
list = scala.collection.immutable.List.apply(1, 2, 3);

インデックスで参照するには

arr(index);
list(index);

これは以下と同じ意味。

arr.apply(index);
list.apply(index);

PHP

arr = array(1, 2, 3);

PHP 5.4からは以下のようにも書ける。

arr = [1, 2, 3];

インデックスで参照するには

arr[index];

Ruby

arr = [1, 2, 3]

インデックスで参照するには

arr[index];

Python

arr = [1, 2, 3]

インデックスで参照するには

arr.index(idx)

JavaScript

下の3つは全部同じ配列を生成する。

arr = [1, 2, 3];
arr = {0: 1, 1: 2, 2: 3}; arr.__proto__ = Array.prototype;
arr = {'0': 1, '1': 2, '2': 3}; arr.__proto__ = Array.prototype;

JavaScript の配列と連想配列の違い
http://d.hatena.ne.jp/amachang/20070202/1170386546

CoffeeScript

arr = [1, 2, 3]
arr = [1, 2, 3, ]
arr = [
    1
    2
    3
]

indexOf

Java

int List#indexOf(Object obj)
int List#lastIndexOf(Object obj)

見つからなかった場合は -1 を返す。

Scala

# Seq[+A]
def indexOf(obj: A): Int
def indexOf(obj: A, from: Int): Int
def indexWhere(p: A => Boolean): Int
def indexWhere(p: A => Boolean, from: Int): Int

見つからなかった場合は -1 を返す。

Ruby

Array#index(obj)

見つからなかった場合は nil を返す。
