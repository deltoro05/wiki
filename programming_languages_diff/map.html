Java, Scala ではマップ、PHPでは連想配列、Rubyではハッシュという。

マップ(連想配列)の書き方

Java

マップは一発では作れない。

import java.util.HashMap;
map = new HashMap();
map.put("a", 1);
map.put("b", 2);

キーで参照するには

map.get("a");

Scala

map = Map("a" -> 1, "b" -> 2);

これは以下のシンタックスシュガー。

map = scala.collection.immutable.Map.apply(scala.Predef.any2ArrowAssoc("a").->(1),
                                           scala.Predef.any2ArrowAssoc("a").->(2));

キーで参照するには

map("a")

Option で取得するには

map.get("a")

PHP

map = array('a' => 1, 'b' => 2);

PHP 5.4からは以下のようにも書ける。

map = ['a' => 1, 'b' => 2];

配列関係の関数 | PHP Manual

Ruby

hash = {'a' => 1, 'b' => 2}

キーで参照するには

hash['a']

Python

map = {'a': 1, 'b': 2}

JavaScript

下の2つは全部同じ連想配列を生成する。

map = {a: 1, b: 2}
map = {'a': 1, 'b': 2}

CoffeeScript

map = {a: 1, b: 2}
map = a: 1, b: 2
map =
    a: 1
    b: 2

isset() は NULL 値を持つ配列キーに対して TRUE を返しません。 一方、 array_key_exists() は TRUE を返します。

Ruby

以下の4つはすべて同じ意味。

Hash#has_key?(key)
Hash#key?(key)
Hash#include?(key)
Hash#member?(key)

clone

JavaScript

shallow copy

function clone(src){
    var dst = {}
    for(var k in src){
        dst[k] = src[k];
    }
    return dst;
}
map_copy = map.clone();

  ...
end

ハッシュの全エントリに対して値のみを引数としてブロックを実行

hash.each_pair do |key, value|
  ...
end
