パターンマッチング

PHP 2013/09/09

preg_match という関数を使い、マッチしたら 1, マッチしなかったら 0 が返される。

var_export(preg_match("/a./", "abc\nabc")); // => 1
var_export(preg_match("/x./", "abc\nabc")); // => 0

preg_match | PHPマニュアル
http://www.php.net/manual/ja/function.preg-match.php

Ruby 2013/09/18

=~ という演算子で、正規表現にマッチしていたら、そのインデックスを返し、マッチしなければ nil を返す。 =~ の左右がPerlと逆。

/a./ =~ "abc\nabc\n" # => 0
/x./ =~ "abc\nabc\n" # => nil

Rubyでは 0 も if の条件の中に入れたら true とみなされる。

-> data_type.md#to-boolean

!~ という演算子は =~ の反対で、正規表現にマッチしていたら false を返し、マッチしなければ true を返す。

/a./ !~ "abc\nabc\n" # => false
/x./ !~ "abc\nabc\n" # => true

Perl 2013/09/18

=~ という演算子で、正規表現にマッチしていれば '1' を返し、マッチしていなければ '' を返す。 =~ の左右がRubyと逆。

print "abc\nabc\n" =~ /a./;
# => 1
print "abc\nabc\n" =~ /a$/;
# => なにも出力しない

!~ という演算子は =~ の反対で、マッチしていれば '' を返し、マッチしていなければ '1' を返す。

正規表現の中にカッコで囲んだグループがあれば、マッチした場合、 $1, $2, $3, … にマッチした文字列が入る。

if文の条件式の中に正規表現だけを書いた場合は、その前に $_ =~ を付けたのと同じになる。

JavaScript 2013/04/19

"abc\nabc\n".match(/a./); // => ["ab"]
"abc\nabc\n".match(/a(.)(.*)/); // => ["abc", "b", "c"]

マッチしない場合は null を返す。マッチした場合は最初にマッチした部分の文字列を配列で返す。パターンにカッコを使った部分があれば配列の2つ目以降としてそれらの部分を返す。
