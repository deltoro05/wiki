Java

配列 2013/09/04

int[] arr = {1, 2, 3};

または

int[] arr = new int[]{1, 2, 3};

java.util.List 2013/09/04

import java.util.List;

List<String> list = new ArrayList<String>();

Scala 2013/10/02

arr = Array(1, 2, 3); // Javaの配列
list = List(1, 2, 3); // scala.collection.immutable.List
list = Nil; // 空のList
list = 1 :: 2 :: 3 :: Nil;

2行目の List は3行目のように :: で区切って書くことも可能。 :: で区切って書く場合は最後が Nil かなんらかの List であることが必要。

上の4行は以下の4行と同じ意味。

arr = scala.Array.apply(1, 2, 3);
list = scala.collection.immutable.List.apply(1, 2, 3);
list = scala.collection.immutable.List.apply();
list = scala.collection.immutable.List.apply(1, 2, 3);

Groovy 2013/10/01

def lst = [1, 2, 3]; // java.util.ArrayList

PHP 2013/09/05

$arr = array(1, 2, 3);

最後に余計なカンマがあっても問題ない。

$arr = array(
             1,
             2,
             3,
            );

PHP5.4以降では以下のようにも書ける。

$arr = [1, 2, 3];

Ruby 2013/09/05

arr = [1, 2, 3]
arr = %w(abc def ghi) # => ['abc', 'def', 'ghi'] と同じ

%w の記法は、カッコは比較的なんでもよくて、以下はすべて同じ意味になる。

['abc', 'def', 'ghi']
%w(abc def ghi)
%w[abc def ghi]
%w!abc def ghi!

Perl 2013/09/23

my @arr = ('abc', 'def', 'ghi');

('abc', 'def', 'ghi') の表記をリストという。リストを @ で始まる変数に代入すると配列になる。

最後に余計なカンマがあっても問題ない。

my @arr = (
           'abc',
           'def',
           'ghi',
          );

配列をネストすることはできず、ネストして書いたつもりでも平坦になってしまう。

my @arr = (1, 2, (3, 4));
print $arr[2]; # => 3
print $arr[3]; # => 4
my @arr2 = (10, 11, @arr);
print "@arr2"; # => 10 11 1 2 3 4

配列へのレファレンス 2013/09/23

my $arr = [1, 2, 3];

で、これは以下と同じ。

my @a = (1, 2, 3);
my $arr = \@a;

レファレンスの実体を取り出すには、

@$arr;
