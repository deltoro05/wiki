配列(リスト)から特定の値の要素をすべて削除するには (remove-by-value)

Java 2013/10/06

以下のメソッドは要素を1つだけ削除する。要素が見つかって削除した場合は true を返し、要素が見つからなかった場合は false を返す。

Signature:

// java.util.List[E]
boolean remove(Object elem);

要素の型が整数の List[Integer] という型の場合は、オートボクシングを期待して、 remove(3) のように呼びだしてしまうと、上記メソッドではなく、 remove(int) という別のメソッドを呼び出してしまう。 lst.remove(new Integer(3)) のようにする必要があるので、要注意。

Scala 2014/10/28

不変(イミュータブル)なリストの類には特定の要素を削除した新しいリストを返すメソッドはないみたい。

例

// seqからelemと等しい要素をすべて削除した新しいリストを返す
seq.filter(_ != elem)

ListBufferなどの可変(ミュータブル)なリストには - や -= がある。

PHP 2014/01/29

例

$elem = 20;

$arr = array(10, 20, 30);
$new_arr = array();
foreach ($arr as $v) {
    if ($v != $elem) {
        $new_arr[] = $v;
    }
}

var_export($new_arr);
// 出漁結果
// array (
//   0 => 10,
//   1 => 30,
// )

Ruby / JRuby 2014/12/31

Array#- を使うと、複数の要素をまとめて全部削除できる。

arr = [10, 20, 30, 40, 50, 30]

p arr - [20, 30]
# => [10, 40, 50]

同じ値の要素が複数あるとすべて削除してくれる。

Python 2014/12/31

リストにある remove メソッドを使う。

Python2/Python3での例

lst = [10, 20, 30, 40, 50, 30]
lst.remove(30)
print(lst)
# => [10, 20, 40, 50, 30]

指定の値を持つ最初の要素を削除する。同じ値の要素が複数あってもすべてを削除してくれるわけではなく、最初の要素のみを削除する。

要素が見つからなかった場合は ValueError: list.remove(x): x not in list というエラーを吐く。
