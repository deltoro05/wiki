メソッド定義

Java 2013/07/10

public String f(String arg1, int arg2){
  ...;
  return ...;
}

返り値がない場合

public void f(String arg1, int arg2){
  ...;
}
  • メソッド呼び出しは、引数のカッコを省略不可

Scala 2013/07/10

def f(arg1: String, arg2: Int): String = {
  ...;
}

返り値が Unit の場合(返り値がない場合)は、以下でも可。

def f(arg1: String, arg2: Int){
  ...;
}
  • メソッド呼び出しは、引数がない場合に限り、カッコを省略できる
  • 関数本体の最後の式が返り値になり return を省略可

PHP 2013/06/09

function f($arg1, $arg2){
  ...;
  return ...;
}
  • 関数定義と呼び出しでは引数のカッコを省略不可

Ruby 2013/04/10

def f(arg1, arg2)
  ...
  return ...
end
  • メソッド呼び出しは引数のカッコを省略可

Perl 2013/07/27

サブルーチンの定義。

sub f {
  my ($arg1, $arg2) = @_;
  ...;
}

サブルーチンの呼び出し。

f($a1, $a2);
  • サブルーチン定義の中では引数は、@_, $_[...] などで参照する
  • サブルーチン呼び出しは、引数がない場合に限り、カッコを省略できる(?)

sh 2013/07/29

関数の定義。

f()
{
    ...
}

関数の呼び出し。

f a1 a2
  • 関数定義の中では引数は $@, $1, $2,,, などを使う
  • 関数呼び出しは通常の外部コマンドの実行と同じくカッコを付けずに引数を並べる

JavaScript 2013/06/09

function f(arg1, arg2){
    ...;
    return ...;
}

または

var f = function(arg1, arg2){
    ...;
    return ...;
}
  • 関数定義と呼び出しでは引数のカッコを省略不可

CoffeeScript 2013/04/28

(arg1, arg2) ->
  ...
  ...

引数がない場合

->
  ...
  ...
  • 関数呼び出しは、引数がある場合はカッコを省略可。引数がない場合はカッコを省略できない。

クロージャ

Ruby 2013/06/09

def builder
  i = 0
  Proc.new do
    puts i
    i = i + 1
  end
end

f1 = builder
f2 = builder
f1.call # 0
f1.call # 1
f2.call # 0
f1.call # 2

Procオブジェクトは、その外側の変数を読み書きできる。

JavaScript 2013/06/09

var builder = function(){
    var i = 0;
    return function(){
        console.log(i);
        i = i + 1;
    }
}
var f1 = builder();
var f2 = builder();
f1(); // 0
f1(); // 1
f2(); // 0
f1(); // 2

メソッドのオーバーライド

Java 2013/07/05

  • メソッドをオーバーライドする場合は、サブクラスのメソッド定義の方に @Override アノテーションを付けたほうがよい(Java 5以降)。 スペルミスなどでオーバーライドできていない場合にコンパイルエラーになってくれる

Scala 2013/07/05

  • メソッドをオーバーライドする場合は、サブクラスのメソッド定義の方に override を付ける
  • 抽象メソッドをサブクラスで定義する場合は、override を付けても付けなくてもよい

PHP 2013/07/18

  • メソッドをオーバーライドするときにつけるキーワードはない。

複数の返り値

PHP 2013/04/22

配列を使って無理やりやろうと思えばできなくもない。

function f(){
  return array(1, "abc");
}
list($num, $str) = f();

Ruby 2013/04/22

return にカンマ区切りで複数の値を書けば、配列を返すことができ、受け取り側は配列の多重代入を使う。

def f
  return 1, "abc"
end
num, str = f

末尾再帰の最適化

Java 2013/07/05

末尾再帰の最適化はしてくれない。

Scala 2013/07/05

末尾再帰の最適化ができるばあいは、コンパイラが自動でそのようにしてくれる。

@scala.annotation.tailrec アノテーションをメソッドに付けると、そのメソッドの再帰呼び出しの箇所が末尾再帰の最適化にならない場合にコンパイルエラーになってくれるので、末尾再帰でないと困る場合はこれを付けたほうが良い。

メソッドがオーバーライド可能だと、末尾再帰できないので、 final も付ける必要がある。
