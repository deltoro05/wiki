  複数行コメント

コメントアウトを一時的に簡単に解除するようにしておくには 2013/04/24

以下のようにすると、1行目を // /* にすることで、コメントアウトの一時的な解除を簡単にできる。

  コメントアウトされたコード

以下のようにすると、1行目を // /* にすることで、コメントアウトのコードと生きているコードを簡単に入れ替えできる。

  コメントアウトされたコード
/*/
  生きているコード
  生きているコード
// */

変数の宣言とスコープ

PHP 2013/08/01

変数の宣言なしにいきなり使うことができる。

関数の中で使用している変数のスコープは関数の中で、関数の外で使用している変数のスコープはグローバルになる。

グローバル変数を関数の中から使用するには、その関数の中で global でグローバル変数を宣言する必要がある。

Ruby 2013/08/04

変数の宣言なしにいきなり使うことができる。

$ で始まる名前の変数がグローバル変数となる。ローカル変数と名前が衝突することはない。

例

a = 1;
# この a はローカル変数

def f1
  # p a
  # ここから外側にある a を参照することはできない

  a = 2
  # このaは関数の中のローカル変数で、外側の a とは無関係

  p a # => 2
end

f1
p a # => 1
# f1 が呼び出されていても f1 の外側の a は変更されない

$a = 10;
def f2
  p $a # => 10
  # $a はどこからでも参照できる

  $a = 20

  p $a # => 20
end

f2
p $a # => 20

Perl 2013/08/01

変数宣言をせずにいきなり値を代入するとグローバル変数になる。
my で宣言すると、ローカル変数になる。

ローカル変数では、{} のブロックの中で宣言すると、ブロックの中のスコープを持つ。

変数名

Ruby 2013/08/04

$ で始まる変数は、グローバル変数。

アルファベット大文字で始まる名前は、定数。

Perl 2013/08/01

$ で始まる変数は、配列やハッシュ(連想配列 or マップ)以外のアトミックな値を保存する変数。
@ で始まる変数は、配列を保存する変数。要素にアクセスするには $...[..] とする。
% で始まる変数は、ハッシュを保存する変数。要素にアクセスするには $...{..} とする。

$, @, % の3つはシジル(sigil)という。

環境変数の取得

Ruby

ENVという変数を利用する。

ENV['HOME']

Perl 2013/07/27

@ENVという変数を利用する。

$ENV{'HOME'}

コマンドライン引数の取得

PHP 2013/05/17

$argc
引数の数。PHPスクリプトのファイル名が最初の引数になるので、これが0になることはない。
http://www.php.net/manual/ja/reserved.variables.argc.php
$argv
引数が配列で入っている。$argv[0] はPHPスクリプトのファイル名になる。
http://php.net/manual/ja/reserved.variables.argv.php

Ruby 2013/05/17

ARGV という定数に引数が配列で入っている。スクリプトファイル名は含まれておらず、先頭から本当の引数が入っている。

Perl 2013/07/31

@ARGV, $ARGV[..] で引数を取得できる。

ソースファイル名と行番号をソースコード中で取得するには

Ruby 2013/05/26

__FILE__

sh 2013/07/05

bashでは、行番号は $LINENO という変数で取得できる。

そのソースファイルが他からの参照ではなく、直接実行された場合にのみ何らかの処理をするには

Ruby 2013/05/26

if __FILE__ == $PROGRAM_NAME
    # 何らかの処理
end

Python 2013/07/30

if __name__ =='__main__':
    # 何らかの処理
