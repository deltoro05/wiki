コメントの書き方

C言語 / C++ 2013/04/22

// 単一行コメント
/*
  複数行コメント
  複数行コメント
*/

複数行コメントはネスト不可。

C言語では、単一行コメントは正式にはC99以降の規格。

Go言語 2013/06/19

// 単一行コメント
/*
  複数行コメント
  複数行コメント
*/

複数行コメントはネスト不可。

Java 2013/10/01

// 単一行コメント
/*
  複数行コメント
  複数行コメント
*/

複数行コメントはネスト不可。

Scala 2013/10/01

// 単一行コメント
/*
  複数行コメント
  複数行コメント
*/

Javaと違い、複数行コメントはネストが可。従って、コメントの中も /* */ の対を対応させる必要がある。

Groovy 2013/10/01

// 単一行コメント
/*
  複数行コメント
  複数行コメント
*/

複数行コメントはネスト不可。

PHP 2013/04/22

// 単一行コメント
# 単一行コメント
/*
  複数行コメント
  複数行コメント
*/

複数行コメントはネスト不可。

http://www.php.net/manual/ja/language.basic-syntax.comments.php

コメントアウトを一時的に簡単に解除するようにしておくには 2013/04/24

以下のようにすると、1行目を // /* にすることで、コメントアウトの一時的な解除を簡単にできる。

/*
  コメントアウトされたコード
  コメントアウトされたコード
// */

以下のようにすると、1行目を // /* にすることで、コメントアウトのコードと生きているコードを簡単に入れ替えできる。

/*
  コメントアウトされたコード
  コメントアウトされたコード
/*/
  生きているコード
  生きているコード
// */

Smarty 2013/09/11

{*
  コメントアウトされたコード
  コメントアウトされたコード
*}

テンプレートの中に <!-- --> を使うと、ブラウザへのレスポンスにもそのコメントが送信されてしまうが、{* *} の形式なら、レスポンスに含まれない。

Python 2013/04/22

# 単一行コメント

複数行のコメントはない。そのかわり、文字列を置いてもプログラムの意味が変わらない箇所で、複数行の文字列リテラルを置くことで、複数行コメントの代わりにすることができる。

'''
複数行コメント
複数行コメント
'''

Ruby / JRuby 2014/10/17

# 単一行コメント
=begin
  複数行コメント
  複数行コメント
=end

文字列や正規表現、%w表記の中など、コメント開始以外の # もありえる。

複数行コメントの =begin, =end は行の先頭に書く必要があり、コメントは =end のある行の行末まで続く。

コメント | Ruby 2.1 リファレンスマニュアル
http://docs.ruby-lang.org/ja/2.1.0/doc/spec=2flexical.html#comment

Perl 2013/12/10

# 単一行コメント
=comment
  複数行コメント
  複数行コメント
=cut

文字列や正規表現の中であったり、$#arr のように、コメント開始以外の # もありえる。

sh (シェルスクリプト) 2015/04/21

# 単一行コメント

複数行コメントの構文はないのだが、複数行のコードを一時的にコメントアウトしたい場合に自分がよく使う方法

cat <<EOF >/dev/null
  複数行コメント
  複数行コメント
EOF
