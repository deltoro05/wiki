関数・メソッド定義 / インデント / セミコロン

PHP

function f($arg1, $arg2){
  ...;
  return ...;
}
  • ブロックはC言語ライクに {} で表現
  • 行末のセミコロンは省略不可
  • 関数呼び出しはカッコを省略不可

Ruby

def f(arg1, arg2)
  ...
  return ...
end
  • ブロックはブロック開始のキーワード(def, do などの開始キーワードと end)で表現
  • 行末のセミコロンは省略可
  • メソッド呼び出しはカッコを省略可

Scala

def f(arg1: String, arg2: Int): String = {
  ...
}
  • ブロックはC言語ライクに {} で表現
  • 行末のセミコロンは省略可
  • メソッド呼び出しは引数のカッコ省略不可だが、引数がない場合の呼び出しはカッコを省略できる

CoffeeScript

(arg1, arg2) ->
  ...
  ...

引数がない場合

->
  ...
  ...
  • ブロックをインデントで表現
  • 行末のセミコロンは省略可
  • 関数呼び出しはカッコを省略可

論理値への変換

Java

boolean型以外が暗黙のうちに boolean に変換されることはない。つまり、false のみが false、true のみが true になる。

Scala

Boolean型以外が暗黙のうちに Boolean に変換されることはない。つまり、false のみが false、true のみが true になる。

PHP

false, null, 0, 0.0, '', '0', array() が false になり、それ以外は true になる。文字列 '0' は false だが、文字列 'false' は true になる。

http://php.net/manual/ja/types.comparisons.php

Ruby

false, nil が false になり、それ以外は true になる。0 も '' も true。

JavaScript

false, 0, '', null, undefined, NaN が false になり、それ以外は true になる。

配列

Java

arr = {1, 2, 3};
arr = new int[]{1, 2, 3};

連想配列は一発では作れない。

import java.util.HashMap;
map = new HashMap();
map.put("a", 1);
map.put("b", 2);

Scala

arr = Array(1, 2, 3);
map = Map('a' -> 1, 'b' -> 2);

これは以下のシンタックスシュガー。

arr = scala.Array.apply(1, 2, 3);
map = scala.collection.immutable.Map.apply(scala.Predef.any2ArrowAssoc('a').->(1),
                                           scala.Predef.any2ArrowAssoc('a').->(2));

PHP

arr = array(1, 2, 3);
map = array('a' => 1, 'b' => 2);

Ruby

arr = [1, 2, 3]
map = {'a' => 1, 'b' => 2}

JavaScript

下の3つは全部同じ配列を生成する。

arr = [1, 2, 3];
arr = {0: 1, 1: 2, 2: 3}; arr.__proto__ = Array.prototype;
arr = {'0': 1, '1': 2, '2': 3}; arr.__proto__ = Array.prototype;

下の2つは全部同じ連想配列を生成する。

map = {a: 1, b: 2}
map = {'a': 1, 'b': 2}

正規表現

Ruby

"abc\nabc\n".gsub(/\n/, "<br />")

2つ目の引数には、かっこでマッチした部分の参照として \1, \2 などが使える。2つ目の引数がダブルクオーテーションで囲まれている場合は \\1, \\2 のようにする必要がある。

"abac".gsub(/a(.)/, "\\1\\1") # => "bbcc"

正規表現のみを与えて、代わりにブロックを与えると、ブロックを評価した結果に置換される。ブロックの中ではかっこでマッチした部分を $1, $2 などで参照できる。

"abac".gsub(/a(.)/) { $1 * 2 } # => "bbcc"

Ruby 1.9.3 リファレンスマニュアル > spec/regexp
http://doc.ruby-lang.org/ja/1.9.3/doc/spec=2fregexp.html

JavaScript

"abc\nabc\n".replace(/\n/, '<br />');
