Encoding::UndefinedConversionError 2013/04/03

String#encode で文字コードを変換したときに、変換できない文字があって Encoding::UndefinedConversionError が発生した場合は、以下のようにオプションを付ける。

str.encode(Encoding::SJIS, :undef => :replace, :replace => "?")

SJISに存在しないUnicode文字があった場合に、これでとりえあずは例外がスローされずに ‘?’ に置き換えられる。

instance method String#encode
http://doc.ruby-lang.org/ja/1.9.3/method/String/i/encode.html

ワンライナー 2013/04/03

-ne をつけると、行ごとに処理するコードをコマンドラインで指定できる。コードの中で $_ は行の文字列が最後の改行付きで入る。以下はなにもせずに、データを聞き流すコード。

cat src.txt | ruby -ne 'print $_'

以下は、改行を削除するコード。

cat src.txt | ruby -ne 'print $_.chomp'

だけど、改行を消すには以下のほうが簡単。

cat src.txt | tr -d '\n'

require と require_relative

Ruby 1.9.2 より前は require で相対パスを指定すれば、カレントディレクトリからたどってファイルを require できたが、 1.9.2 以降はそれができなくなった。require のサーチパスに以前は '.' が入っていたのだが、セキュリティの都合で1.9.2以降は '.' が外されたため。

ただし、1.9.2以降でも require './hoge' というようにパスを './' から書けば、カレントディレクトリからたどってくれる。

さらにただし、 '.' というのは実行時のカレントディレクトリであって、 requireの呼び出し元のファイルの場所ではない。

呼び出し元のファイルの場所からの相対で指定するには、新たに追加されたメソッドである require_relative './hoge' を使う。これは呼び出し元のファイルがあってこそなので、eval の中などでは使えない。

Kernel.#require_relative(relative_feature)
http://rurema.clear-code.com/version:1.9.3/query:Kernel.%23require_relative/
