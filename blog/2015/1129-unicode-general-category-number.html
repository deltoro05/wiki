Unicode文字プロパティのNumber 2015/11/29

正規表現で数字にマッチさせたい場合に \p{N} というのを使うことがあるが、これにマッチする文字は [0-9] に限らない。 \p{N} はUnicodeの文字プロパティで N (Number)とされているすべてのコードポイントにマッチする。これはとてもたくさんの文字が該当する。

Rubyでの例

str = "8٨८೮൮๘໘༨၈៨᠘８𝟖𝟠𝟪𝟴𝟾Ⅷⅷ༱⁸₈⅛⑧⑻⒏⓼❽➇➑㈧㊇🄉八"
p str.scan(/\p{N}/)  # => ["8", "٨", "८", "೮", "൮", "๘", "໘", "༨", "၈", "៨", "᠘", "８", "𝟖", "𝟠", "𝟪", "𝟴", "𝟾", "Ⅷ", "ⅷ", "༱", "⁸", "₈", "⅛", "⑧", "⑻", "⒏", "⓼", "❽", "➇", "➑", "㈧", "㊇", "🄉"]

微妙に太さや装飾の違う “8” がたくさんあるが、フォントの違いではなくUnicodeのコードポイントがすべて異なる。 “8” に見えないよくわからない外国の文字もたくさんある。

漢数字の “八” を括弧や丸で囲んだものも \p{N} でマッチするが、単なる “八” はマッチしない。

N (Number) は正確には Nd (Decimal_Number), Nl (Letter_Number), No (Other_Number) の 3種類ある。正規表現でもそれぞれ \p{Nd}, \p{Nl}, \p{No} と表記し、 \p{N} は [\p{Nd}\p{Nl}\p{No}] と同じである。

この3種類それぞれすべての文字を以下にリストアップしてみた。

  • Decimal_Number
  • Letter_Number
  • Other_Number
